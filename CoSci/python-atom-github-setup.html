atom

To install atom, go to atom.io and hit download. Default settings should be fine.

P.S.: Your parents might have a text editor they think is cooler. Ask them about it! There's lots of different ones out there, and different ones are better for different kinds of programming.

python

  1. Download the latest stable version of Python 3
  2. Follow the prompts to install. If the installer asks whether you want to add Python to PATH, click "Yes"
  3. To run a program, cd to the appropriate directory (remember, cd .. is to go up a directory) and then run "python {name of program}.py"
  4. BONUS: You can test small bits of code with the python interpreter. Your parents will know about this, but basically you can set variables and interact with the just like in a real program, but in real time, similar to the Roblox command line. Great for testing out a new library or checking some math.

github

  1. Download and install the latest stable version of git. MAKE SURE YOU SELECT ATOM AS YOUR DEFAULT TEXT EDITOR. Or something else, just don't let it pick VIMS. A lot of developers swear by VIMS, but it's really hard to use. If you showed me a computer with VIMS on it I literally would not know how to close the program or open a new file. Lots of keyboard shortcuts to memorize. Only use it if you're ready.
  2. Once you have git installed, search for git in the windows search bar and open the "git bash".
    3. Setting Up a Project for the First Time
    1. Go to your project's github page and click "clone or download". Copy the URL that shows up to your clipboard.
    2. Use cd to get to the directory you want to put the project in (I like to make a new folder for each of my projects).
    3. Run "git init" in the directory for your project.
    4. In the same directory, run "git pull {your project url here}"
    5. BONUS: To open your project in Atom from the command line, run "atom ." from inside your project folder. You can also do this for an individual file by typing "atom {filename}" (as long as you're in the file's directory)
    6. BONUS #2: If you are making a NEW project that isn't on your github yet, follow all the above steps, except that 1. You first need to make an empty repository on github 2. Instead of "git pull {your project url}", run "git remote add origin {your project url}". This tells git (the program on your computer) where on Github to look for your program.
    4. Pushing Your Changes to Github
    1. Go to your project directory.
    2. Run "git add ." and "git commit -m '{describe your changes in a short sentence}'". This will add all your changes to the git registry and give them the magic stamp of approval. You can commit multiple times before uploading to github if you'd like.
    3. Finally, run "git push origin master". You will need to log in to your github account. If it doesn't work, try running "git remote add origin {url}"

Have fun!
